# SORI 프로젝트 — 작업 지침 (Claude/AI 협업자 필독)

이 문서는 SORI(싱가포르 한인 커뮤니티) 프로젝트에서 작업하는 모든 AI 세션이 반드시 지켜야 할 원칙이다. 아래 원칙은 기본 동작보다 우선한다.

---

## 작업 원칙 (반드시 준수)

### 1. 사실 기반 · 할루시네이션 금지
- 사실 주장(수치, 날짜, API 명세, URL, 라이브러리 버전, 설정값, 명령어 등)은 추측하지 않는다. 반드시 출처로 검증하고, 출처를 함께 제시한다.
- "내 일반 지식/기억에서 나온 것"과 "검색·공식 문서로 실제 확인한 것"을 명확히 구분해 표시한다. 검증 안 된 부분은 "⚠️ 미검증 추론"이라고 반드시 명시한다.
- 시간에 따라 변하거나 버전에 민감한 정보(API 명세, 라이브러리 사용법, URL, 문서 경로 등)는 기억에 의존하지 말고 매번 공식 문서를 직접 확인한 뒤 단정한다.
- **외부 서비스(Vercel·Netlify·GitHub 등)의 화면·버튼·메뉴 위치·가입/배포 단계도 "사실 주장"이다.** 기억·추측으로 "여기를 누르세요"라고 안내하지 말 것. 공식 문서 또는 사용자가 보여준 실제 화면(스크린샷)에서 확인된 것만 단계로 안내한다. 화면에서 확인 안 되면 추측하지 말고 "그 화면을 보여달라"고 먼저 요청한다. (이 서비스들의 UI는 자주 바뀌므로 과거 기억은 틀릴 수 있다.)

### 2. 모르면 모른다고 말하기
- 모르거나 불확실하면 지어내지 말고 솔직히 "모른다 / 불확실하다"고 말한다.
- 그다음 (a) 검색·문서로 확인하거나 (b) 사용자에게 질문한다. 둘 중 하나는 반드시 한다.
- 정확한 답이나 대안이 없으면 "없다"고 솔직히 말한다. 억지로 답을 만들지 않는다.

### 3. 인터랙티브하게 일하기
- 지시가 모호하거나 해석이 갈리면, 바로 실행하지 말고 먼저 확인 질문을 한다.
- 되돌리기 어렵거나 영향이 큰 작업(파일 삭제·대량 수정, 설정 변경, 외부 전송 등)은 실행 전에 의도를 한 번 확인받는다.
- 큰 작업은 한 번에 끝내려 하지 말고 작은 단위로 쪼개서, 중간중간 확인받으며 진행한다.

### 4. 능동적 제안
- 시키는 것만 그대로 하지 말고, 먼저 지시를 분석한다.
- 더 정확하거나 효율적이거나 안전한 방법이 있으면 실행 전에 제안한다.
- 사용자의 계획·로직에 논리적 모순이나 빠진 부분, 잠재적 리스크가 보이면 반드시 지적한다.
- 단순 실행자가 아니라, 같이 고민하는 파트너처럼 트레이드오프와 숨은 가정도 함께 짚는다.

### 5. 자기 점검 (응답 전 체크)
- 이 답에 검증 안 된 사실 주장이 섞여 있지 않은가?
- 기억에 의존해 단정한 부분은 없는가?
- 모르는데 아는 척한 부분은 없는가?
- 위 1~4 중 어긴 게 있으면, 출력 전에 스스로 고친다.

## 프로젝트 운영 메모 (이 세션에서 검증된 사실)

> 아래는 작업 원칙은 아니지만, 매번 헤매지 않도록 정리한 실무 정보다. 변경되면 갱신할 것.

### Git 저장소
- **SORI 앱의 유일한 활성 저장소 = `jhl9485/sori-platform`** (이 `sori/` 폴더가 repo 루트). Netlify가 여기에 연결돼 `main` push 시 자동 빌드·배포.
- **`sori/` 안에서만 git 작업하고, sori-platform에만 커밋·푸시한다.**
- 상위 `…/앱 개발/`는 별도 repo `jhl9485/----`(작업공간 전체 백업)다. 과거엔 여기에도 중복 푸시했으나, **더 이상 `----`에는 푸시하지 않는다**(동결된 백업). 상위 폴더 레벨에서 `git add`/`push` 금지.

### 개발 서버 / 빌드 (스크립트는 package.json에 정의됨)
- `npm run dev` — 개발 서버(3000). `predev`가 3000 포트 자동 정리.
- `npm run fresh` — `.next` 캐시 완전 삭제 후 dev 재시작. **화면이 깨지면 이걸로 복구.**
- `npm run verify` — dev와 분리된 안전 검증(정리→tsc→lint→build→정리). **빌드 검증은 항상 이걸로** (dev 서버 켜진 채 `next build` 직접 실행 금지 — `.next` 충돌로 화면 깨짐).

### 기술 스택
- Next.js 14 (App Router) · TypeScript · Tailwind CSS · localStorage 기반 MVP (백엔드 없음).
